Journal

5 August

  • Nic and Will brainstormed ideas for the battle bot design.
  • Compared prebuilt chassis vs. custom build options.
  • Decided to go with a custom design with additional features from prebuilt chassis for more flexibility in weapon and motor integration.
  • Will used CAD modeling tool Onshape to visualize potential designs and design a chassis.
  • Matt started a python and html code for the website.
  • Aware that custom designs would take more time and effort but provide better learning opportunities.

7 August

  • Continued refining the design concept, focusing on motor placement and weight distribution.
  • Used Onshape drawings to finalize initial design sketches.
  • Decided to prioritize flexibility in motor and sensor integration.
  • Matt continued with the website code using Flask to build a backend and HTML and CSS for the frontend.

12 August

  • Will worked on detailed CAD drawings for the battle bot's structural design.
  • Nic worked on analysing 
  • Matthew analyzed compatibility between Python, Microbit, and other hardware components.
  • Identified a potential issue with the power supply for the motors.
  • Revised wiring and battery configurations to ensure the motors received adequate power and to the right pins.
  • Concerns about balancing time spent on design vs. time for coding and building.

14 August

  • Further refined the CAD designs, adjusting motor placement for better balance.
  • Matthew confirmed Microbit makecode was the best platform for hardware integration.
  • Spent time analyzing power requirements and motor wiring.
  • Decision to move forward with building and coding, though time management remained a concern.

19 August

  • Matthew began coding the motor control using Python and Microbit makecode.
  • Initial testing was to see if the motor could be turned on and off, but wiring errors caused the motor to malfunction.
  • Nic and Matthew researched open-source libraries for motor control and maybe sensor integration.
  • Matthew coded a gradual motor speed control but couldn’t test it due to wiring and voltage issues.
  • Realized the project would face delays unless wiring were corrected.
  • Matthew collaborated with Alex Mospan and decided to change the website to be hosted online using github.

26 August

  • Will and Matthew started assembling the battle bot, focusing on motor placement and chassis stability.
  • Encountered issues with motor alignment, which made the bot unstable; had to adjust the chassis design.
  • Electrical wiring errors caused problems with current flow to the motor and microbit.
  • Nic continued researching coding libraries and resources to help streamline coding efforts.
  • Delays due to wiring issues meant progress on coding and testing was slow.
  • Matthew removed all python code for the backend and decided to focus more on HTML and CSS code to make the website look better.

28 August

  • Matthew and Will spent 1.5 periods soldering wires for other groups.
  • The motor still wasn’t functioning due to bad wiring configuration.
  • The motor was receiving insufficient current and the battery was underperforming, affecting performance.
  • Nic began working on the project website, documenting progress and challenges.
  • Frustrations were mounting due to the ongoing wiring problems, and time pressure was becoming a concern.
  • Matt finished off the website design.

2 September

  • Matthew simplified the motor control code to troubleshoot one step at a time.
  • Will helped test the motor, discovering that inconsistent current flow was the main issue.
  • Nic continued developing the website, documenting the project and research findings.
  • Realized the main issue was a lack of expertise with electrical wiring, delaying both the coding and hardware testing.
  • Worked on motor alignment, but the wiring problem persisted.
  • Nic and Matt started working on the Gantt chart and filling it in.

4 September

  • Matthew and Will continued working on the wiring, trying to fix current flow problems that were preventing the motor from working properly.
  • Simplified code further to focus on motor control, but issues with wiring were still hampering progress.
  • Nic worked on refining the website, documenting the ongoing challenges and coding process.
  • Motor performance was inconsistent, but small improvements were made with each adjustment.

9 September

  • Will and Matthew spent most of the day soldering and adjusting the wiring.
  • Finally managed to fix the wiring, allowing the motor to run more consistently, though some performance issues remained.
  • Code started functioning correctly after wiring adjustments.
  • Nic finalized most of the website content, including descriptions of the coding process, research, and challenges.
  • Limited time left for additional sensor integration or performance tweaks, but the motor was now functional.

11 September

  • Matthew and Will focused on final tweaks to improve motor performance and signal consistency.
  • The motor and code were working together, though there were still minor speed inconsistencies.
  • Nic completed the website, documenting the entire project, including the wiring and coding challenges.
  • Time pressure was significant, but the project was nearing completion.
  • Soldering and wiring fixes were critical to the final success of the motor control, though it left less time for optimizing other areas.

16 September

  • Matt started designing the weapon of the battle Bot to be laser cut.
  • Will started redesigning the chassis of the bot to be laser cut.
  • Nic continued refining the Gantt chart and adding more details to the paperwork such as website analysis.
  • Concerns about the finish date of the paperwork.
  • Will figured out that the issue with the voltage was a loose cable touching the wrong terminal on the motor controller.
  • Will and Matt fixed the wiring issue.

18 September

  • Nic Lim added some things to the logbook.
